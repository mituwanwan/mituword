## 🚀 快速开始

### 1️⃣ 第一步：安装 Docker（如果没装）

```bash
# 运行安装脚本
./scripts/setup.sh

# 如果脚本需要密码，可以手动执行官方安装命令：
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 把当前用户加入docker组（避免每次sudo）
sudo usermod -aG docker $USER
newgrp docker  # 重新加载用户组

# 验证安装
docker --version
```

### 2️⃣ 第二步：启动开发环境

```bash
# 启动MySQL + phpMyAdmin
./scripts/start.sh

# 你应该看到：
# - MySQL 在 3306 端口
# - phpMyAdmin 在 8080 端口
```

### 3️⃣ 第三步：初始化应用

```bash
# 1. 安装依赖（替换PostgreSQL为MySQL）
npm uninstall pg @prisma/adapter-pg
npm install mysql2

# 2. 生成Prisma客户端
npx prisma generate

# 3. 同步数据库结构
npx prisma db push

# 4. 启动开发服务器
npm run dev
```

## 🎯 日常运维命令

### 🔧 启动与停止

| 命令 | 作用 |
|------|------|
| `./scripts/start.sh` | 启动数据库和工具 |
| `./scripts/stop.sh` | 停止服务（保留数据） |
| `./scripts/logs.sh` | 查看实时日志 |
| `./scripts/logs.sh db` | 只看MySQL日志 |

### 💾 备份与恢复

| 命令 | 作用 |
|------|------|
| `./scripts/backup.sh` | 备份当前数据库 |
| `./scripts/restore.sh` | 恢复到最新备份 |
| `./scripts/restore.sh backups/xxx.sql.gz` | 恢复到指定文件 |

### ⚠️ 清理与重置

| 命令 | 作用 |
|------|------|
| `docker compose down` | 停止并删除容器（保留数据） |
| `./scripts/destroy.sh` | 完全销毁（删除数据） |

## 🐳 Docker 基础学习

### 常用命令

```bash
# 查看正在运行的容器
docker ps

# 查看所有容器（包括停止的）
docker ps -a

# 查看镜像
docker images

# 进入容器内部（调试用）
docker exec -it my-app-mysql bash

# 查看资源占用
docker stats

# 查看容器日志
docker logs my-app-mysql -f  # -f 是跟踪模式
```

### Docker Compose 常用

```bash
# 启动所有服务
docker compose up -d

# 只启动某些服务
docker compose up -d db phpmyadmin

# 停止服务
docker compose stop

# 停止并删除
docker compose down

# 重新构建并启动
docker compose up -d --build
```

## 📖 运维学习要点（详解）

### 要点1：为什么要用 Docker？

**传统方式：**
- 手动安装MySQL，版本混乱
- 每台机器手动配置环境
- 换个电脑要重新搞一遍
- "我这里明明是好的" 😂

**Docker方式：**
- 配置文件里写好，一行命令启动
- 所有环境完全一致
- 换个电脑同样能用
- 想删就删，不污染系统

### 要点2：Volume - 数据持久化

看我们 docker-compose.yml 中的配置：
```yaml
volumes:
  - mysql_data:/var/lib/mysql
```

作用：即使容器销毁，数据还在 Volume 里！下次启动容器，数据恢复。

### 要点3：Network - 服务之间的通信

在 Docker Compose 中，服务名就是域名！
- 在 app 容器里，可以用 `db` 作为MySQL的主机名
- 不需要IP地址，Docker网络帮你搞定！

### 要点4：健康检查

```yaml
healthcheck:
  test: ["CMD", "mysqladmin", "ping", "..."]
```
这样可以确保数据库准备好之后，才启动应用服务！

## 🔧 故障排查手册

### 问题1：端口被占用
```bash
# 查找占用端口的进程
sudo lsof -i :3306

# 如果是MySQL占用，可以停止：
sudo systemctl stop mysql
sudo systemctl disable mysql  # 禁用开机启动
```

### 问题2：权限被拒绝
```bash
# 可能是用户没在docker组里
sudo usermod -aG docker $USER
# 然后重新登录，或运行：
newgrp docker
```

### 问题3：连不上数据库
```bash
# 检查容器是否在运行
docker ps

# 查看MySQL日志
docker logs my-app-mysql

# 进入容器手动测试
docker exec -it my-app-mysql mysql -u myapp_user -p
```
