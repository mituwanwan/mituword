# 迷途世界 (MituWorld) — 风险评估

> 版本：v1.0  
> 日期：2026-06-04  

---

## 1. 技术风险

### 1.1 网易云API不稳定（高风险）

**风险描述**
- NeteaseCloudMusicApi 是逆向工程项目，非官方API
- 网易云可能随时更改接口加密逻辑
- 个人服务器IP可能被风控

**影响**
- 音乐板块完全不可用
- 用户体验严重受损

**应对方案**
1. **独立部署**：与主站隔离，更新时不影响其他功能
2. **监控告警**：定时健康检查，API异常时发通知
3. **优雅降级**：API不可用时显示"音乐服务维护中"，不影响其他功能
4. **缓存策略**：歌单信息缓存24小时，减少API调用
5. **备选方案**：如果API长期不可用，改为展示歌单列表+跳转网易云

**责任人**：开发 + 运维

---

### 1.2 主题样式冲突（中风险）

**风险描述**
- 双主题涉及大量条件样式
- 可能出现某个主题下组件显示异常
- 第三方组件（如有）可能不支持主题切换

**影响**
- 视觉体验不一致
- 品牌形象受损

**应对方案**
1. **统一组件库**：所有UI走自研组件，禁止硬编码颜色
2. **视觉回归测试**：每次修改后检查双主题效果
3. **CSS变量方案**：使用 CSS 自定义属性，切换时只改根变量

---

### 1.3 字体加载性能（低风险）

**风险描述**
- ZCOOL KuaiLe 约 150KB，Press Start 2P 约 20KB
- 网络慢时可能出现字体闪烁（FOUT）

**影响**
- 像素主题首屏加载稍慢
- 字体切换时闪烁

**应对方案**
1. `font-display: swap` 避免阻塞渲染
2. 预加载关键字体 `<link rel="preload">`
3. 使用系统字体作为fallback

---

## 2. 版权与合规风险

### 2.1 音乐版权（高风险）

**风险描述**
- 网易云VIP歌曲外链播放存在版权灰色地带
- 虽然API能获取直链，但这不等于有分发授权

**影响**
- 理论上存在被网易云投诉的可能
- 实际个人网站流量小，风险极低

**应对方案**
1. **不存储音频文件**：只存储元数据（歌名、封面、链接）
2. **明显提示**：页面标注"音频来自网易云音乐，仅供个人分享"
3. **跳转优先**：优先引导用户去网易云客户端播放
4. **及时响应**：如收到通知，立即移除播放功能改为纯展示

---

### 2.2 分享网站内容合规（低风险）

**风险描述**
- 好站分享中的链接可能指向有问题的网站

**影响**
- 极小，但需防范

**应对方案**
1. 所有分享链接添加 `rel="noopener noreferrer"`
2. 后台添加时人工审核
3. 仅分享知名/可信网站

## 3. 运维风险

### 3.1 服务器资源不足（中风险）

**风险描述**
- 2核2G 同时跑 Next.js + MySQL + 网易云API + Nginx
- 高峰期内存可能吃紧
- MySQL 默认配置占用较多内存

**影响**
- 服务响应变慢
- OOM 导致进程被杀

**应对方案**
1. **MySQL调优**：
   ```ini
   innodb_buffer_pool_size = 256M
   key_buffer_size = 32M
   ```
2. **Swap分区**：添加 2G Swap 作为缓冲
3. **监控**：部署后观察内存使用，必要时升级 2核4G
4. **自动重启**：Docker 容器设置 `restart: always`

---

### 3.2 数据丢失（中风险）

**风险描述**
- 服务器故障导致数据库丢失
- 误操作删除数据

**影响**
- 所有内容丢失

**应对方案**
1. **定时备份**：每日凌晨自动备份 MySQL
   ```bash
   mysqldump -u root -p mituworld > /backup/mituworld_$(date +%Y%m%d).sql
   ```
2. **保留7天备份**：自动清理旧备份
3. **本地开发环境保留副本**

---

### 3.3 HTTPS证书过期（低风险）

**风险描述**
- Let's Encrypt 证书 90天有效期
- 忘记续期导致网站无法访问

**应对方案**
1. 使用 `certbot` 自动续期
2. 设置日历提醒

## 4. 项目进度风险

### 4.1 需求蔓延（中风险）

**风险描述**
- 开发过程中不断添加新功能
- 导致原定排期无法完成

**应对方案**
1. **冻结需求**：每个 Week 开始前确定需求范围，Week 中不加新功能
2. **二期清单**：新想法记入"二期需求池"，不在本期做
3. **MVP原则**：先让核心功能可用，再追求完美

## 6. 应急预案

### 场景：网易云API突然不可用

```
1. 检测：每5分钟健康检查 /health
2. 通知：API异常时发送通知（邮件/钉钉）
3. 降级：音乐板块自动切换到"展示模式"（只显示歌单，不播放）
4. 修复：检查开源项目issue，更新API镜像
5. 恢复：验证后切回完整模式
```

### 场景：服务器宕机

```
1. 阿里云控制台重启实例
2. Docker Compose 自动重启容器
3. 检查日志定位原因
4. 如有数据问题，从备份恢复
```
