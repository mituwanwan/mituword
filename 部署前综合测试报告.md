- **建议**：生产环境使用 Redis 等分布式存储替代内存 Map。

## 五、安全测试结果

| 检查项 | 风险等级 | 结果 |

### 5.2 XSS (跨站脚本攻击)

| 检查项 | 风险等级 | 结果 |
|--------|----------|------|
| Markdown 渲染 | 低 | `react-markdown` 默认禁止原始 HTML，无 `rehypeRaw` |
| 链接协议过滤 | 低 | `javascript:`, `data:`, `vbscript:`, `file:` 已被拦截 |

| 日记摘要展示 | 低 | `excerpt` 和 `content.substring` 通过 React 渲染，自动转义 |

### 5.3 权限控制与认证

| 检查项 | 风险等级 | 结果 |
|--------|----------|------|
| 管理后台路由保护 | 中 | `middleware.ts` 检查 `ADMIN` 角色，正确 |

| 注册速率限制 | 低 | 每小时5次，合理 |
| API 通用限流 | 低 | 每分钟60次读取/30次写入，合理 |

## 七、问题汇总与优先级

### 7.1 高危问题 (1项)

| ID | 问题 | 影响 | 修复建议 |
|----|------|------|----------|
| **H1** | API 权限控制缺陷：仅验证登录状态，不校验 ADMIN 角色 | 任何注册用户可执行管理员操作（增删改项目/日记/音乐等） | 在所有管理类 API 的 `session` 检查后增加 `session.user.role === 'ADMIN'` 校验；或在 `rateLimitResponse` 后统一添加权限检查中间件 |

### 7.2 中危问题 (4项)

| ID | 问题 | 影响 | 修复建议 |
|----|------|------|----------|
| **M1** | 缺失核心功能：好站分享 (/sites) | PRD 中 P0 功能未实现，影响产品完整性 | 按 PRD 实现 SiteShare 的前台展示页和后台管理功能 |
| **M2** | 缺失核心功能：每日日报 (/daily) | PRD 中 P0 功能未实现，影响产品完整性 | 按 PRD 实现 DailyReport 的展示、自动生成和管理功能 |
| **M3** | API 输入验证缺失：projects/diaries/music 的 POST/PUT 未使用 Zod | 可传入非法字段、格式错误数据，甚至利用 Prisma 的嵌套写入特性进行未授权操作 | 为所有写操作 API 添加 Zod Schema 验证，与前端表单验证保持一致 |
| **M4** | 速率限制基于内存存储 | 多实例/容器化部署时限流失效，可能导致暴力破解或 API 滥用 | 使用 Redis 替代内存 Map 存储限流状态 |

### 7.3 低危问题 (8项)

| ID | 问题 | 修复建议 |
|----|------|----------|
| **L1** | 缺少 `.env.test` 导致单元测试失败 | 创建 `.env.test` 文件并配置测试数据库连接 |
| **L2** | Playwright 仅配置 Chromium | 增加 Firefox 和 WebKit 测试项目，覆盖更多浏览器 |
| **L3** | Nginx 缺少 HTTPS 配置 | 配置 SSL 证书、443端口、HTTP 自动跳转 HTTPS |
| **L4** | 缺少 Content-Security-Policy 头 | 在 Nginx 或 Next.js headers 中配置合适的 CSP |
| **L5** | 缺少 Strict-Transport-Security 头 | HTTPS 部署后配置 HSTS |
| **L6** | 密码复杂度策略较弱 | 注册 Schema 中增加密码复杂度要求（大小写+数字+特殊字符） |
| **L7** | 缺少系统主题偏好检测 | 在 `useTheme` 初始化时读取 `window.matchMedia('(prefers-color-scheme: dark)')` |
| **L8** | 缺少动态 Sitemap 和 JSON-LD | 创建 `/app/sitemap.ts` 和结构化数据组件 |

## 九、测试结论
